# Detecting Human vs. AI-Generated News: A Text Analytics Approach Using NLP Techniques

Final assignment for the **Natural Language Processing and Text Analytics** course at Copenhagen Business School (Spring 2025).

This project investigates the linguistic and structural differences between AI-generated and human-written political news (headlines and short descriptions), then builds classifiers to detect AI-generated text. It compares two feature-based models (Multinomial Naive Bayes and Logistic Regression) against a transformer-based BERT model, and evaluates BERT's robustness on an out-of-distribution (OOD) dataset.

## Research questions

The project answers, in three parts:

1. Which descriptive, lexical, syntactic, sentiment, and topic features differ significantly between AI-generated and human-written text?
2. How effective are traditional machine learning models at classifying AI- vs. human-written text using these features?
3. How does feature-based model performance compare to a transformer-based BERT classifier?

## Key findings

- **Distinguishing features:** AI-generated text tends to be longer, more repetitive in word choice and sentence structure, more negative in tone (VADER), and thematically narrower than human-written text.
- **Feature-based models:** Logistic Regression reached up to **78% accuracy** (Naive Bayes 76%), both performing best on combined headline + description input.
- **BERT:** Significantly outperformed the feature-based models with **98% accuracy** on in-domain data and **85%** on the OOD (Entertainment) evaluation set — showing strong but reduced cross-domain generalization.

## Data

The data is in `300525_Final_Data_NLP.zip`:

| File | Description |
|------|-------------|
| `News_Category_Dataset_v3.json` | Source dataset: ~210,000 HuffPost headlines + short descriptions, 2012–2022 (Misra, 2022) |
| `political_2017.csv` | Final combined dataset — human (filtered to Politics, 2017) + AI-generated, shuffled and labelled |
| `llama_huffpost_dataset.csv` | Synthetic politics dataset generated with LLaMA 2 7B chat (one-shot prompting) |
| `llama_huffpost_dataset_entertainment.csv` | Synthetic OOD evaluation dataset (Entertainment, zero-shot prompting) |

The human subset was filtered to **Politics articles from 2017** (before GPT-1's 2018 release, to ensure human authorship). The synthetic data was generated by prompting LLaMA 2 7B chat via Hugging Face, matched to the human text's style and length.

## Pipeline overview

The notebook runs end-to-end:

1. **Datasets** — load and filter the HuffPost data; generate synthetic politics and entertainment datasets with LLaMA.
2. **EDA** — category and per-year distributions, word-count analysis.
3. **Preprocessing** — spaCy (`en_core_web_md`): lowercasing, stop-word and non-alphabetic removal, lemmatization, POS tagging, NER.
4. **Cleaning** — remove empty rows and duplicates (final dataset: 20,590 rows).
5. **Feature engineering** — five levels: descriptive, lexical (POS, NER, special characters, function/content ratio, TF-IDF, type-token ratio), structural (n-grams, sentence starters, conjunctions, repetitiveness), sentiment (VADER, TextBlob), and topic modelling (LDA). Produces a 125-column feature dataset.
6. **Classification** — Multinomial Naive Bayes, Logistic Regression, and BERT, each on headline / description / combined inputs, plus BERT's OOD evaluation.

## Usage

1. Unzip `300525_Final_Data_NLP.zip` so the CSV/JSON files are available to the notebook.
2. Open `300525_Final_Notebook_NLP.ipynb` (Colab recommended for GPU access).
3. Run cells top to bottom. Note: the synthetic dataset generation step (LLaMA on A100) took ~7.5 hours — the generated CSVs are included in the zip so you can skip regeneration and run the analysis directly.

## Notes & limitations

- The synthetic data carries common LLM-generation limitations (self-amplified bias, limited diversity, mode collapse). The OOD evaluation dataset used zero-shot prompting because one-shot prompting caused the model to duplicate the example.
- The synthetic dataset was built solely to evaluate NLP detection methods; factual accuracy and broader ethical implications of AI-generated news were outside the project's scope.
